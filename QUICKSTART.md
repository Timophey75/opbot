# 🚀 БЫСТРЫЙ СТАРТ (5 минут)

Не хочется читать все документы? Вот минимум что нужно для начала!

## ⚡ БЫСТРО НА ПК/МАК

```bash
# 1. Установка (потребует ~3 мин)
bash setup.sh

# 2. Запуск (в 2 разных терминалах)
# Терминал 1:
python web_app.py

# Терминал 2:
python bot.py
```

✅ Готово! Откройте http://localhost:5000 в браузере

## 🔑 ВХОД

- **Код администратора:** `0000`
- **Для операторов:** генерируется при создании

## 📱 ПЕРВОЕ ИСПОЛЬЗОВАНИЕ

### Как админ:

1. Входите кодом `0000`
2. Нажимаете "Операторы" → "Добавить оператора" 
3. Вводите имя и фамилию оператора
4. **Сохраняете код оператора** (4 цифры)
5. Идете в "Календарь" 
6. Нажимаете на день → "Создать мероприятие"
7. Вводите название и время
8. Нажимаете "Назначить" и выбираете оператора ✅

### Для оператора:

1. Получают 4-значный код от админа
2. Открывают приложение
3. Вводят свой код
4. Видят свое расписание в календаре
5. Могут отказаться от смены если нужно

## 🌐 НА ИНТЕРНЕТЕ (Бесплатно)

### Вариант 1: Render (Рекомендуется)

1. GitHub → new repo → загружаете opbot
2. https://render.com → new web service
3. Выбираете opbot с GitHub
4. Добавляете переменные:
   ```
   TELEGRAM_BOT_TOKEN=your_token
   ADMIN_CODE=0000
   ```
5. Deploy ✅

**Время:** ~5 мин

### Вариант 2: Railway (Еще проще)

1. https://railway.app → Deploy now
2. Выбираете GitHub repo
3. Добавляете variables
4. Готово! ✅

**Время:** ~3 мин

## 📱 НА ANDROID (Termux)

```bash
# 1. Установка Termux из F-Droid
# 2. В Termux:

pkg update && pkg upgrade
pkg install python pip git
git clone https://github.com/yourusername/opbot.git
cd opbot
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python init.py

# 3. Запуск
python web_app.py &
python bot.py &
```

Откройте http://localhost:5000 в браузере телефона

## 🚨 ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ

### "Python не найден"
```bash
python --version  # Должно быть 3.8+
```

### "Модуль не найден"
```bash
pip install -r requirements.txt
```

### "Порт 5000 занят"
Отредактируйте `web_app.py`, измените `port=8000`

### "Не открывается приложение"
- Убедитесь оба скрипта запущены (bot.py и web_app.py)
- Проверьте http://localhost:5000 в браузере
- Посмотрите логи в консоли
